Glossary

Definitions of acronyms and technical terms used across the Publik ecosystem, with concrete examples.

API

Application Programming Interface — a documented set of endpoints, parameters and responses that allow software to communicate in a predictable way.

Publik example: A WCS workflow step sends citizen data to a city CRM via a Passerelle endpoint that accepts JSON and returns a result code.

Design endpoints with clear resource names and HTTP verbs; validate inputs; return explicit error details.

Back-office

Restricted administrative interface used by staff/agents to process requests, configure services and access operational data.

Publik example: An agent opens the back‑office, reviews a submitted form, adds a note and moves the case to the next status.

BI

Business Intelligence — turning operational data into charts, tables and indicators to support decisions.

Publik example: Monthly counts of submitted requests per service are displayed to managers to track workload.

BI/OLAP

Combined practice of Business Intelligence (dashboards) and OLAP (multidimensional aggregation) on structured data.

Publik example: Analysts explore submissions by month, service and status to spot trends and bottlenecks.

BIJoe

Publik’s analytics layer that queries data organized for analysis and produces results for dashboards.

Publik example: A dashboard shows the number of WCS submissions closed each week, computed by BIJoe.

Cell

Reusable content block placed in portal pages; can be static text/media or dynamic data fetched from services.

Publik example: A 'My Requests' cell displays the logged‑in user’s latest submissions.

Chrono

Publik’s module for managing agendas, available slots and bookings for appointments or events.

Publik example: Citizens pick a passport appointment slot; agents adjust opening hours in the agenda.

CLI

Command‑Line Interface — text-based commands executed in a terminal to administer or automate tasks.

Publik example: An administrator runs a command to anonymize old bookings across tenants.

CMS

Content Management System — software to create pages, menus, layouts and publish content without coding.

Publik example: Editors build a citizen portal homepage by placing cells and editing text.

Combo

Publik’s portal used by citizens (front‑office) and agents (back‑office) to access services and content.

Publik example: The portal shows a tile linking to a form and a cell listing the user’s ongoing requests.

Connector

Adapter that encapsulates authentication, protocol and field mapping for a specific external system.

Publik example: A connector fetches a list of streets from a municipal database and returns it in a simplified format.

Include robust error handling and timeouts; log requests and responses for traceability.

CRM

Customer Relationship Management — system used to track interactions with citizens and manage requests or contacts.

Publik example: A submitted form creates or updates a contact record in the city’s CRM through an integration.

Cron

Time-based scheduler that runs tasks at fixed times or intervals on Unix-like systems.

Publik example: A nightly job exports anonymized statistics to the analytics database.

CRUD

Create, Read, Update, Delete — four fundamental operations on stored data.

Publik example: An agent creates a new agenda entry, views it, updates the time window, or deletes it if unused.

CSV

Comma‑Separated Values — plain‑text table where each row is a record and fields are separated by a delimiter.

Publik example: Back‑office users export submissions as CSV to review them in a spreadsheet.

DB

Database — structured storage engine used by applications to persist and query data.

Publik example: Operational data and analytics are stored in databases designed for their respective workloads.

Django

High‑level Python web framework with an ORM, templating and admin interface.

Publik example: Portal pages and business models are implemented with Django components.

DSL

Domain‑Specific Language — compact syntax focused on a narrow problem domain.

Publik example: Conditions in a workflow use a small expression language to enable or block transitions.

EOPayment

Payment abstraction library that provides a unified way to initiate and validate transactions with multiple payment providers.

Publik example: A payment operation is started and later confirmed using the same high‑level API regardless of the provider.

ERP

Enterprise Resource Planning — central system handling finance, HR, logistics and related processes.

Publik example: An integration retrieves an invoice balance from the city’s ERP before offering online payment.

ETL

Extract, Transform, Load — move data from source systems, reshape it, and load it into a store optimized for analysis.

Publik example: Form submissions are periodically exported, normalized and loaded into analytics tables.

Fargo

Publik’s module for storing and circulating documents produced or uploaded during procedures.

Publik example: A generated PDF decision letter is saved and later retrieved from the document store.

FranceConnect

National identity federation that lets users sign in to public services with trusted credentials.

Publik example: The portal offers a 'Sign in with FranceConnect' option during authentication.

Front-office

Public‑facing part of the platform used by citizens.

Publik example: A citizen opens the homepage, browses services and submits an online request.

GDPR

General Data Protection Regulation — EU rules governing personal‑data processing and protection.

Publik example: Old appointment records have personal fields removed after the configured retention period.

GI

Geographic Information — data that describes locations, shapes or attributes of real‑world features.

Publik example: A dataset of street names and coordinates is used to prefill addresses in a form.

GIS

Geographic Information System — software that stores and analyzes geographic data and exposes it via services.

Publik example: A map service provides the list of street segments for an address autocomplete field.

GUI

Graphical User Interface — windows, forms, buttons and menus used to operate software visually.

Publik example: Agents use a web GUI to filter requests and perform actions on cases.

HA

High Availability — architectural practices that minimize downtime via redundancy and failover.

Publik example: Two application nodes serve the portal behind a load balancer; a shared storage holds uploaded files.

Hobo

Publik’s orchestration portal that manages tenants and shared configuration across modules.

Publik example: An administrator creates a new tenant; configuration is propagated to the relevant services.

Hobo-agent

Background agent that executes operations and scheduled jobs as instructed by the orchestration portal.

Publik example: A command is run across all tenants to refresh analytical exports.

HR

Human Resources — domain covering employee data, positions and schedules.

Publik example: An integration checks staff availability from an HR system before assigning a task.

iCal

Internet calendaring data format used to represent events and schedules.

Publik example: A calendar feed is exposed so staff can subscribe to appointment agendas.

ICS

File extension (.ics) for iCalendar files that can be imported into calendar applications.

Publik example: A booking confirmation email includes an .ics attachment so the citizen adds it to their calendar.

ID

Identifier — a unique value used to reference a record, request or object.

Publik example: Each submission has an ID shown to the user for follow‑up and used internally for queries.

JSON

JavaScript Object Notation — lightweight text format for structured data composed of objects and arrays.

Publik example: A connector returns {"street": "Rue de...", "zip": "35000"} which the form uses to prefill fields.

Lingo

Publik’s module for listing amounts due, initiating online payments and recording results.

Publik example: A parent pays a canteen invoice online; the associated case is marked as paid.

MDX

Multidimensional Expressions — query language used by some OLAP engines to retrieve measures by dimensions.

Publik example: A cube of submissions could be queried to count cases per month and status.

Multitenancy

Single deployment serves multiple isolated tenants so each organization’s data and configuration are separated.

Publik example: Two cities use the same platform on different subdomains with separate data stores.

NFS

Network File System — protocol for sharing directories over a network so multiple servers access the same files.

Publik example: Uploaded documents are stored on shared storage that all application nodes can reach.

OAuth

Authorization framework where an application obtains a token to access an API on a user’s behalf without handling the password.

Publik example: A connector calls an external service with a bearer token obtained during a login flow.

OCR

Optical Character Recognition — process of extracting text from images or scanned documents.

Publik example: A workflow step processes an uploaded scan to detect a reference number automatically.

ODS

OpenDocument Spreadsheet — open standard spreadsheet format.

Publik example: Back‑office users export case lists to ODS for editing in LibreOffice.

OIDC

OpenID Connect — authentication layer on top of OAuth 2.0 that issues standardized identity tokens.

Publik example: A portal relies on an identity provider that returns an ID token after successful login.

OLAP

Online Analytical Processing — fast aggregation and slicing of data along multiple dimensions.

Publik example: Counts of requests by department and month are computed for a dashboard.

ORM

Object–Relational Mapping — technique that maps tables and rows to classes and objects in code.

Publik example: A model 'Request' maps to a database table; queries return objects rather than raw rows.

Passerelle

Publik’s integration hub that standardizes access to external systems via configurable connectors.

Publik example: A form autofills address data by calling a connector instead of querying the external API directly.

PCI

Payment Card Industry Data Security Standard and related rules that govern handling of cardholder data.

Publik example: Card data is handled by the payment provider; the platform stores only transaction references.

PDF

Portable Document Format — fixed‑layout document format suitable for archiving and printing.

Publik example: A decision letter is generated as a PDF and stored for later download.

PostgreSQL

Open‑source relational database known for reliability, strong SQL features and extensions.

Publik example: Analytical queries run against tables organized for reporting in a PostgreSQL schema.

PSP

Payment Service Provider — company that processes online payments on behalf of merchants.

Publik example: The payment flow redirects the user to a PSP page and later confirms the result.

QR

Quick Response code — 2D barcode that encodes text or URLs for quick scanning.

Publik example: A receipt embeds a QR code linking to a verification page for that transaction.

Quixote

Lightweight Python web framework focused on straightforward page composition.

Publik example: Legacy form flows rely on components historically implemented with this framework.

REST

Style of web APIs where resources are identified by URLs and manipulated with HTTP methods like GET, POST, PUT and DELETE.

Publik example: A GET request retrieves an agenda; a POST creates a booking.

SAML

Security Assertion Markup Language — XML standard for exchanging authentication and authorization data between identity and service providers.

Publik example: After successful login at the identity provider, a signed assertion is posted to the portal to establish the session.

SCOP

Société Coopérative et Participative — French worker cooperative legal form where employees collectively own and govern the company.

Publik example: In a cooperative model, contributors participate in governance and share long‑term responsibility for the software.

SDK

Software Development Kit — tools, libraries and documentation provided by a vendor to integrate with their systems.

Publik example: When connecting to a payment provider, a developer may use the provider’s SDK to sign requests and parse responses.

SMS

Short Message Service — text messaging over mobile networks.

Publik example: A notification step can send a text message through a configured SMS gateway.

SMTP

Simple Mail Transfer Protocol — standard protocol for sending email.

Publik example: The platform sends confirmation emails using the organization’s configured SMTP server.

SOAP

XML‑based web service protocol that wraps messages in an envelope with headers and a typed body.

Publik example: A connector builds a SOAP request to query a legacy registry and parses the XML response.

SQL

Structured Query Language — language used to define tables and query relational data.

Publik example: A report counts requests per day with a GROUP BY on the submission date.

SSO

Single Sign‑On — user authenticates once and then accesses multiple applications without re‑entering credentials.

Publik example: A citizen logs in and reaches portal pages and forms without another login prompt.

Star Schema

Data‑warehouse design with a central fact table linked to dimension tables such as date, category or location.

Publik example: Each submission becomes a fact row linked to a date dimension for monthly totals.

TLS

Transport Layer Security — protocol that encrypts network connections and authenticates servers (and optionally clients).

Publik example: APIs and portals are accessed over HTTPS to protect credentials and data in transit.

Trigger

HTTP endpoint designed to advance or modify a case when called with the right parameters and authorization.

Publik example: After a successful payment, a call sets the related case status to “Paid”.

UI

User Interface — the visual components that users interact with: pages, forms, buttons, messages.

Publik example: The portal’s UI lets citizens navigate services and submit requests.

URL

Uniform Resource Locator — the address of a resource on the web (scheme, host, path, optional query).

Publik example: The form page has a unique URL that can be bookmarked or shared when appropriate.

UX

User Experience — overall ease, efficiency and satisfaction a person has when using a system.

Publik example: Clear labels and simple steps reduce errors and speed up form completion.

w.c.s.

Historical spelling of the WCS module name; refers to the same component.

Publik example: Older documentation may use “w.c.s.” while current interfaces display “WCS”.

WCS

Publik’s form builder and workflow engine for creating online procedures and processing submissions.

Publik example: A city publishes a form; agents process the resulting requests through configured statuses and actions.

Webhook

HTTP callback sent by a system to notify another system that an event has occurred.

Publik example: A payment provider posts a notification to confirm a transaction outcome.

Workflow

Configured sequence of statuses, transitions and actions that define how a case is processed from start to finish.

Publik example: A request moves from “Waiting” to “Open”, triggers an email, then reaches “Done” once validated.

XML

Extensible Markup Language — text format that represents structured data with nested elements and attributes.

Publik example: A legacy service responds with an XML document that the connector parses into fields used by a form.

XMLA

XML for Analysis — protocol used by some analytical servers to expose multidimensional data to client tools.

Publik example: An external BI tool can query cubes built from exported submissions through an XMLA endpoint.

YAML/JSON

Two structured data formats: YAML is indentation‑based and human‑friendly; JSON uses braces and brackets and is ubiquitous in web APIs.

Publik example: A configuration file defines connector parameters in YAML; a response from a service is consumed as JSON.
